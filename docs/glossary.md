# Ghost Glossary

A reference of key Ghost terminology, abbreviations, and backward-compatibility notes.

## Terms (A–Z)

**Agent** — The pi-agent-core reasoning loop that processes user prompts, calls tools, and produces responses. See [Architecture: Runtime Composition](./reference/architecture.md)

**Agent Loop** — The continuous cycle: user message → orchestrator.prompt() → agent.prompt() → tool calls → response → persist. See [Architecture: Runtime Composition](./reference/architecture.md)

**Allowlist** — Approved Telegram user IDs in the `channel_allowlist` table; users not on the list receive a pairing challenge. See [Architecture: Security & Policy](./reference/security.md)

**Autonomy Level** — Permission tier (read_only, interactive, supervised, full) that gates tool/command execution. See [Architecture: Security & Policy](./reference/security.md)

**Channel** — A communication adapter (Telegram, web, CLI) that publishes inbound messages to the bus and consumes outbound responses. See [Architecture: Channels](./reference/channels.md)

**Cloid** — Client order ID. Ghost generates deterministic cloids for paper-trading orders using SHA-256 hash of orderId. See [Operations: Paper Trading](./operations/paper-trading.md)

**Confirm Card** — Modal on the web dashboard showing trade details (side, size, SL, TP) for user approval of high-risk operations. See [Architecture: Runtime Composition](./reference/architecture.md)

**Confirmable Tools** — 8 trading tools that require explicit user approval before execution: place_order, cancel_order, cancel_all_orders, emergency_close, set_sl_tp, bracket_order, partial_close, adjust_margin. See [Architecture: Security & Policy](./reference/security.md)

**ContextBuilder** — Factory that assembles the system prompt layer-by-layer: SOUL.md + memory + skills + runtime context. Called on every prompt turn. See [Architecture: Memory & Skills](./reference/memory-and-skills.md)

**Cron Job** — Scheduled task (stored in `~/.ghost/cron.json`) that fires on a schedule and executes via the runner. See [Operations: Scheduler](./operations/scheduler.md)

**Dispatcher** — Consumes inbound messages from the bus, gates concurrency via semaphore (default 5), calls orchestrator, manages streaming state machine, retries with exponential backoff. See [Architecture: Dispatcher](./reference/architecture.md)

**EventBus** — 1:N pub/sub for services to emit events (price alerts, wallet updates) to subscribers (web, Telegram). See [Architecture: Runtime Composition](./reference/architecture.md)

**FIFO MessageBus** — FIFO queue (inbound + outbound) that decouples channels from the orchestrator. No topics; channels identified by (channel, chatId) tuple. See [Architecture: Runtime Composition](./reference/architecture.md)

**Gateway** — ElysiaJS HTTP server on port 15401 (default) hosting WebSocket, REST endpoints, web dashboard, and approval card delivery. See [Architecture: Overview](./reference/architecture.md)

**Ghost Cloid** — Cloid generated by Ghost during paper trading; prefixed to identify Ghost-placed orders in attribution. See [Operations: Paper Trading](./operations/paper-trading.md)

**HyperliquidClient** — Facade wrapping the @nktkas/hyperliquid SDK, abstracting live wallet, order, and position queries. Wrapped by PaperTradingClient in paper mode. See [Architecture: Overview](./reference/architecture.md)

**Intel Service** — Market intelligence aggregator: liquidation maps, funding rates, cross-exchange basis, whale tracking. See [Architecture: Overview](./reference/architecture.md)

**Judge** — LLM reasoning call (via runner) that evaluates observer events and determines whether to emit a user-facing notification. See [Operations: Observer](./operations/observer.md)

**Leak Detector** — Security scanner that scrubs credentials (API keys, private keys, JWTs, DB URLs) from agent outputs in real-time. See [Architecture: Security & Policy](./reference/security.md)

**MEMORY.md** — Long-term memory file storing facts, trader preferences, and resolved issues. Always injected into system prompt. See [Architecture: Memory & Skills](./reference/memory-and-skills.md)

**HISTORY.md** — Append-only timestamped log of consolidated message chunks; archive for grep-based recall. See [Architecture: Memory & Skills](./reference/memory-and-skills.md)

**MessageBus** — FIFO work queue with separate inbound/outbound channels; no topics. See [Architecture: Runtime Composition](./reference/architecture.md)

**Observer** — Unified proactive scanner on dual cadences: EVAL (5s, in-memory) and SYNC (60s, REST poll) that detects fills, liquidations, canceled orders, and fires alerts. See [Operations: Observer](./operations/observer.md)

**Orchestrator** — Session-lock gatekeeper that serializes prompts per session, consolidates memory if needed, swaps agent context, builds system prompt, refreshes tools, and persists messages. See [Architecture: Runtime Composition](./reference/architecture.md)

**Paper Trading** — Simulation engine using live market data; SQLite-backed, persistent across restarts, with configurable balance and margin tiers. See [Operations: Paper Trading](./operations/paper-trading.md)

**PaperEngine** — Core state machine simulating order fills using mark prices from PriceCache; tracks balances, positions, orders, fills. See [Operations: Paper Trading](./operations/paper-trading.md)

**Pairing** — Device authentication flow: unauthorized users receive an 8-char code, paste it in web dashboard, get approved to allowlist, then can use Telegram. See [Architecture: Channels](./reference/channels.md)

**Persona** — Trader archetype (e.g., aggressive scalper, conservative hodler) used in eval and prompt tuning. See [Architecture: Overview](./reference/architecture.md)

**Pre-Trade Advisory** — Skill that gathers portfolio state, funding, technicals before an order and outputs risk level + suggested SL/TP. See [Architecture: Memory & Skills](./reference/memory-and-skills.md)

**RC Channel** — Release Candidate channel (vs. latest). Users can opt into pre-release builds for early testing. See [Operations: Update](./operations/update.md)

**Runner** — Background job orchestrator using taskAgent with single-flight promise chain to serialize news summarization, eval, cron delivery. See [Architecture: Runtime Composition](./reference/architecture.md)

**Runtime** — Composition root (src/runtime.ts) that wires 17+ subsystems in dependency order and returns a Runtime object. Entry point: createRuntime(). See [Architecture: Runtime Composition](./reference/architecture.md)

**Session** — Persistent JSONL message history per channel+chatId, loaded on demand, consolidated when token budget overflows. See [Architecture: Runtime Composition](./reference/architecture.md)

**Skill** — Markdown file with YAML frontmatter that injects behavior guidance into the system prompt. Builtin skills ship with Ghost; workspace skills override by name. See [Architecture: Memory & Skills](./reference/memory-and-skills.md)

**Skill Loader** — Service that syncs filesystem skill files with DB state, discovers new skills, and surfaces them for enable/disable toggling. See [Architecture: Memory & Skills](./reference/memory-and-skills.md)

**SOUL.md** — System prompt seeded to workspace on first daemon start; user-editable master instructions. See [Architecture: Memory & Skills](./reference/memory-and-skills.md)

**Three-Layer Model** — Feature design split: Capability (US: what data in/out) → Behavior (Skill: how to phrase) → Eval (measure). Never put behavior in US. See [Architecture: Memory & Skills](./reference/memory-and-skills.md)

**Tool Registry** — Registry of 40+ tools (generic + trading) with TypeBox parameter schemas. Refreshed on every prompt to surface late-registered tools. See [Architecture: Overview](./reference/architecture.md)

**TypeBox** — Schema validation library used for all tool parameter schemas in Ghost. See [Architecture: Overview](./reference/architecture.md)

**Watchlist** — User's watched tickers stored in DB; surfaced in context builder. See [Architecture: Overview](./reference/architecture.md)

**Alert Rule** — Persistent price condition (e.g., BTC > $75k) stored in DB; observer checks rules on each EVAL tick. See [Operations: Observer](./operations/observer.md)

**Notification** — Outbound alert sent to configured channels (Telegram, web) when an observer event passes the judge filter. See [Architecture: Overview](./reference/architecture.md)
